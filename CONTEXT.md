# Project Context — AI Shorts Content Pipeline

Automated pipeline that generates YouTube Shorts content using Gemini AI + Pexels stock assets.
One video per day, 7 genres rotating by day of the week.

---

## Pipeline Flow

```
Idea → Script → Characters → Scenes → Visual Search Queries → Pexels Search → Download Assets → Asset Manifest
```

Each story step produces a JSON file saved to `data/`.
Downloaded assets land in `data/assets/`.
An asset manifest records which Pexels assets were chosen for debugging and reruns.
All assets share a traceable ID lineage:

```
IDEA-DRM-20260614-001
  └── SCR-DRM-20260614-001
        ├── CHAR-DRM-20260614-001
        └── SCN-DRM-20260614-001          (scene file — contains all 5 scenes)
              ├── SCN-DRM-20260614-001-01  → data/assets/SCN-DRM-20260614-001-01.jpg/.mp4
              ├── SCN-DRM-20260614-001-02  → data/assets/SCN-DRM-20260614-001-02.jpg/.mp4
              ├── SCN-DRM-20260614-001-03  → data/assets/SCN-DRM-20260614-001-03.jpg/.mp4
              ├── SCN-DRM-20260614-001-04  → data/assets/SCN-DRM-20260614-001-04.jpg/.mp4
              └── SCN-DRM-20260614-001-05  → data/assets/SCN-DRM-20260614-001-05.jpg/.mp4
        └── MANIFEST-DRM-20260614-001.json (records chosen Pexels assets per scene)
```

## File Reference

### `src/index.ts`

Entry point. Runs the full pipeline sequentially in 13 steps.

```
bootstrap()   async — orchestrates all pipeline steps in order
```

**Pipeline steps:**
1. `IdeaGenerator.generate()`
2. `ScriptGenerator.generate(idea)`
3. `CharacterGenerator.generate(script)`
4. `SceneGenerator.generate(script, characters)`
5. `VisualSearchGenerator.generate(scenes)`
6. `pexels.searchPhotos(query)` or `pexels.searchVideos(query)` per scene based on preferredMediaType
7. `downloader.download(sceneId, url)` per scene
8. Generate asset manifest with chosen Pexels assets
9. `VoiceGenerator.generate(script)` — narration audio via Gemini TTS
10. `CaptionGenerator.generate(voiceFile)` — caption segments + SRT file
11. `Renderer.render(manifest, sceneMap, voiceFile, captionFile)` — final MP4
12. `MetadataGenerator.generate(idea, script, videoFile)` — publishing metadata
13. `publishVideo(videoFile, metadataFile)` — upload to YouTube as private

### `src/modules/media/caption.ts`

Step 10 — Splits narration into timed caption segments and writes an SRT file.
No transcription, no audio upload. The narration text is already known from the voice step.
Gemini splits the text into 3–8 word chunks and distributes timestamps across the audio duration.
SRT formatting is pure TypeScript — no AI involved.

**Approach:** narration text + audio duration → Gemini → segments → `toSrt()` → `.srt` file

**Interfaces**

```
interface CaptionSegment
  text    string   caption line (3–8 words)
  start   number   seconds
  end     number   seconds

interface CaptionFile
  id        string   CAP-GEN-DATE-SEQ
  scriptId  string
  audioId   string   links back to AUD-* for the same run
  title     string
  segments  CaptionSegment[]
  srtPath   string   absolute path to .srt file
  createdAt string
```

**Internal**

```
buildPrompt(narration, duration)   instructs Gemini to split text into 3–8 word segments
                                   spanning the full audio duration, max 4s per segment
toSrt(segments)                    converts segments to SRT format — pure arithmetic, no AI
formatSrtTime(seconds)             e.g. 8.5 → "00:00:08,500"
```

**Exports**

```
class CaptionGenerator
  generate(voiceFile)   async → CaptionFile
```

**Storage:**
- JSON: `data/media/captions/CAP-*.json`
- SRT:  `data/assets/captions/CAP-*.srt`

### `src/modules/youtube/auth.ts`

One-time OAuth setup script. Run manually once per channel:

```
npx tsx src/modules/youtube/auth.ts
```

Uses a `localhost:3000` redirect URI — **not** the deprecated `urn:ietf:wg:oauth:2.0:oob`.

**Before running, add this to your OAuth client in Google Cloud Console:**
APIs & Services → Credentials → your OAuth Client ID → Authorized redirect URIs:
```
http://localhost:3000/callback
```

Flow:
1. Starts a temporary HTTP server on `localhost:3000`
2. Opens the Google authorization URL in the browser automatically (`start ""`)
3. You log in and approve access
4. Google redirects to `localhost:3000/callback` — the script catches the code automatically
5. Exchanges the code for tokens, prints the refresh token
6. Server shuts down

No copy-pasting required. Copy the printed refresh token into `.env`:
```
YOUTUBE_REFRESH_TOKEN=your_token_here
```

If no refresh token is returned, revoke app access at `myaccount.google.com/permissions` and run again.
