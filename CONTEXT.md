# Project Context — AI Shorts Content Pipeline

Automated pipeline that generates YouTube Shorts content using Gemini AI + Pexels stock assets.
One video per day, 7 genres rotating by day of the week.

---

## Pipeline Flow

```
Idea → Script → Characters → Scenes → Visual Search Queries → Pexels Search → Download Assets → Asset Manifest
```

Each story step produces a JSON file saved to `data/`.
Downloaded assets land in `data/assets/`.
An asset manifest records which Pexels assets were chosen for debugging and reruns.
All assets share a traceable ID lineage:

```
IDEA-DRM-20260614-001
  └── SCR-DRM-20260614-001
        ├── CHAR-DRM-20260614-001
        └── SCN-DRM-20260614-001          (scene file — contains all 5 scenes)
              ├── SCN-DRM-20260614-001-01  → data/assets/SCN-DRM-20260614-001-01.jpg/.mp4
              ├── SCN-DRM-20260614-001-02  → data/assets/SCN-DRM-20260614-001-02.jpg/.mp4
              ├── SCN-DRM-20260614-001-03  → data/assets/SCN-DRM-20260614-001-03.jpg/.mp4
              ├── SCN-DRM-20260614-001-04  → data/assets/SCN-DRM-20260614-001-04.jpg/.mp4
              └── SCN-DRM-20260614-001-05  → data/assets/SCN-DRM-20260614-001-05.jpg/.mp4
        └── SCN-DRM-20260614-001.manifest.json (records chosen Pexels assets per scene)
```

---

### `src/pipeline/hydrate-context.ts`

```
hydrateContext(run: RunRecord): Promise<PipelineContext>
```

Rebuilds context from disk for single-step retry:
1. Loads idea and script from `run.steps[...].outputId`
2. Derives all downstream IDs from `ctx.script.id` using id-generator functions
3. Loads manifest from `data/assets/SCN-*.manifest.json`
4. Logs a hydration status line showing ✓/✗ per field

### `src/api/pipeline.routes.ts`

Three endpoints:

**POST /pipeline/start**
- Calls `runService.getActiveRun()` — returns 409 if a run is already running/pending
- Creates run, responds 202, runs pipeline in background

**POST /pipeline/step/:step**
- Validates step name and runId
- Hydrates context from disk via `hydrateContext(run)`
- Runs the single step in background

**POST /pipeline/cancel/:runId**
- Calls `cancellationRegistry.cancel(runId)` — flips the in-memory token so the pipeline loop stops at the next inter-step boundary (current step still finishes)
- Also marks the `RunRecord` as failed on disk immediately so the dashboard reflects the intent without waiting for the current step to complete
- If the run is not in this process's registry (e.g. server restarted mid-run), only the disk update happens

### `src/api/server.ts`

Express 5 on port 3001. CORS enabled. Static MP4 files served:
```typescript
app.use("/videos", express.static(path.join(process.cwd(), "output", "videos")));
```

`assetsRouter` is mounted **only** at `/assets`. The `/videos` static middleware is registered
before the asset router so there is no path shadowing — `GET /videos/:filename` serves MP4 files
and `GET /assets/videos` serves the JSON metadata collection.

        ├── AUD-HOR-20260615-001

        ├── VID-HOR-20260615-001
